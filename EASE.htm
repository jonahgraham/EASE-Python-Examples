Exercise 1: Getting started

Welcome to the EclipseCon France 2016 tutorial on EASE & Python!

This document contains the exercises and instructions. To be able to run through the exercises you will need to have followed the setup instructions from https://github.com/jonahkichwacoders/EASE-Python-Examples.

Hello World

Launch Eclipse, switch to the Scripting perspective and ensure you have a Py4J script shell open.

Using the Py4J Script Shell view, run hello world, by pasting the following in. The output will appear in the Console view.

print("Hello World")

The console also allows you to paste into the input box, try to copy and paste this line.

print("Question: Can we extend Eclipse with Python?\nAnswer: Yes, we can")

TIP: You can drag and drop commands to the script shell results section.

Try some mathematical operations:

2+2

7*8

TIP: Use the up and down arrows in the script shell input box to show history. History can also be accessed from the dropdown arrow on the script shell input area.

The Py4J Script Shell also supports multiline input. Try the lines below in the script shell. Note you must ensure you add the whitespace at the beginning of the 2nd line.

for line in range(10):

print("I love scripting")

 

Not adding the space will result in an IndentationError being raised such as this:

 

File "<console>", line 2

print("I love scripting")

^

IndentationError: expected an indented block

 

Python gives you access to a wealth of libraries. Use import to access a library

import math

math.sqrt(2)

OR

from math import sqrt

sqrt(9)

Check which version of Python you are using with the sys library

import sys

sys.version

sys.platform

sys.path

 

Once you are done, feel free to let us know by pasting your sys.version into the EASE channel on Mattermost!

Exercise 2: EASE modules

In addition to Python libraries, EASE provides modules. Modules are logical groupings of functionality such as UI, Launch or Debug. Modules typically package up Eclipse and Java APIs to allow them to be scripted from Python or any other language supported by EASE.

UI module

One popular module for controlling the Eclipse user interface is the UI module. Give it a try from the Py4J Script shell.

loadModule("/System/UI")

showInfoDialog("Hello World")

The available modules can be browsed using the Modules Explorer view. Try this code:

showView("Modules Explorer")

maximizeView("Modules Explorer")

The Modules Explorer view provides the following features:

        Browse modules and functions

        Right-click links to help

        Drag to script shell to load them

Modules must be loaded before being used. They can be loaded in any of these ways:

        By using the loadModule command in the script shell

        By using the loadModule drop down button in the script shell

        By dragging a module from the Modules Explorer to the script shell results pane

Resources module

This gives you functions to access the Eclipse resources such as Projects (IProject), Files (IFile & File), Folders, etc. The top level functions can be browsed in the Modules explorer.

The API for using specific objects will correspond to the Eclipse Java APIs. For example to see what methods you can use with an IProject, refer to the Eclipse Platform API Specification:

        http://help.eclipse.org/mars/index.jsp?topic=%2Forg.eclipse.platform.doc.isv%2Freference%2Fapi%2Foverview-summary.html

Exercise 3: Executing EASE Scripts

Besides the script shell, we can also run scripts directly. If you haven't already, create a workshop project with a file called hello.py. Copy these contents to the file and save the file:

loadModule("/System/UI")

name = showInputDialog("What is your name?")

 

if name is not None:

showInfoDialog("Welcome " + name)

Right click on the file and select 'Run As > EASE Script'. This should popup the first input dialog followed by the info dialog if a name is provided.

EASE Script Explorer

Scripts can also be run from the Script Explorer view. This shows all the scripts discovered based on the 'Script Locations' preference setting.

1. Click on scripts view

2. Click on 'Preferences button'

3. In the 'Script Preference' dialog page click Add Workspace.

4. Add the 'workshop' project. Click OK

5. Click OK to close the dialog.

6. The 'hello.py' script should now show up in the Script Explorer. Select it and press the green arrow to run it.

Script Locations could be in the workspace, on the file system or even a URI for instance to point to GIT.

Exercise 4: Magic Keyword Headers

EASE supports magic keyword headers that are used to hook scripts into the IDE framework. A full list can be found here: https://wiki.eclipse.org/EASE/Scripts

Key

Description

name

Script name displayed. May use / to create folder structures. Defaults to the filename.

description

Script description. Used as a tooltip on the script.

toolbar

Bind a script to a view toolbar. Provide either view ID or view title (eg Project Explorer).

menu

Bind a script to a view menu. Provide either view ID or view title (eg Project Explorer).

popup

Bind a script to a popup menu. Add an expression like enableFor(full.qualified.class.name). If the selection is an instance from that type or adaptable to it, the popup menu will be added.

keyboard

Binds a script to a keyboard shortcut. Syntax for keyboard mappings is the same as in the 'Keys' preferences page.

image

Add a dedicated toolbar/menu image. Accepts local file paths, relative paths (according to the script file) or URIs. Accepted URI schemes are http;//, platform:/plugin, workspace://, file://

Exercise 4.1: Add magic keyword headers to hello.py to create a button that shows up in the Project Explorer view.

Exercise 6: Quickly Extend Eclipse

Python scripting in EASE can be used quickly extend the current Eclipse runtime without the need for Java code, building or installing plug-ins. This can be very handy for rapid prototyping of new features in script code. The following exercises give examples of features that can be implemented to quickly extend the IDE.

These exercises can be found in the 'Extend' project of the examples repository.

Explore From Here

Write a script that adds functionality to the right-click pop-up menu of the Project Explorer. The functionality should work on resources and allow users to bring up the Explorer program on the selected resource. You can follow the instructions in Extend/ex3_explorer.py

Private To Public

Write a script that, given a text selection, converts any 'private' keywords to 'public'. You can follow the instructions in Extend/ex4_find_replace.py

To test, you can use utilities from PyUtilities to create 5 java projects each of which contains a Java file with some private variables.

Exercise 8: Share Functionality

Python scripting allows you to easily share functionality with scripts. In this way scripts could be used to share configuration of the IDE (e.g. preferences) or even steps to reproduce a bug. As scripts can also be run from the command line, they can also be integrated with build and test set-ups. These exercises are designed to highlight scripts that share functionality.

Setup Preferences

Write a script that sets up the Eclipse preferences for an improved IDE experience. You can follow the instructions in Share/ex7_setup_preferences.py.

exercise 9: A simple EASE module

Create a new Plug-in project called org.eclipse.tutorial.curvefitting. We do neither need UI contributions nor an Activator. In the Plug-in Manifest editor switch to the Extensions tab and add an extension of type org.eclipse.ease.modules. Add a new module to it, provide a name and a class implementation.

The class should be a POJO providing 2 simple methods:

int add (int first, int second);

int decrement(int first, int second);

When finished try out your class by opening the plugin.xml file and clicking the run button in the upper right corner. In the launched application switch to the Scripting perspective and try out your module.

exercise 10: Annotations & Dependencies

For more control on exported methods & fields use @WrapToScript annotations. They can be used for methods and static fields. Without them, all public methods will be exported.

Exported methods may support optional parameters by using parameter annotations:

public void myFunction(@ScriptParameter(defaultValue=) String myParameter, )

EASE tries its best to cast the string value to the desired parameter type.

Exercise: let the 2nd parameter of decrement default to 1.

Dependencies to other modules are defined at the module definition. They can easily be accessed when the module derives from org.eclipse.ease.modules.AbstractScriptModule:

getEnvironment().getModule(SomeModule.class);

Exercise: create a dependency to the Charting module and provide a method:

private ChartingModule getChartingModule();

In your own module.

Extract the doclet to a local folder. Then run javaDoc (Menu/Project) on your module project.

Doclet class path: C:\...\EclipseCon_France\Tutorial\org.eclipse.ease.helpgenerator\bin

-root C:\...\EclipseCon_France\Tutorial\eclipse\workspace\org.eclipse.tutorial.curvefitting
